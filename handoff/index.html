Rules the code must enforce

  • Blue = interactive only. Primary buttons, links, active nav, the In-Progress dot. Never on trip IDs, service tags, or proof-doc icons.
  • Verified replaces Completed. A single State column derived from both fields via getStateStyle — not two chips side by side.
  • One colored container per panel. Slide-out forms (Cancel, Reject, Remark) are mutually exclusive; the active one is the only tinted box.
  • Amber is text-only on data rows. No tinted backgrounds for Awaiting Verify rows — a dot + text reads cleaner.
  • No row tints for exception states. Cancelled/Rejected surface via the state cell + muted subline, not a pink row.

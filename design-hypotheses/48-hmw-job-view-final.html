HMW-48: Job View — Final Design

"Definitive mockup for the Jobs page incorporating unified status lifecycle (Pending → In Progress → Completed → Verified) at 30-vendor scale."
Consolidates decisions from HMW-44 through HMW-47:
• Flat power table as default (not vendor grouping) — scales to 30+ vendors
• Unified job status: Pending → In Progress → Completed (proof uploaded) → Verified (admin sign-off)
• Status pills as primary workflow navigation, not proof-status pills
• Group by toggle (None/Vendor/Service/Date) — vendor grouping as secondary lens
• Rejected jobs surface via red rows within Active view, not as separate pill
• No document rejection feature — proof lifecycle is one-way

View 1: Active (Default)

Default on page load
All jobs that need attention. Active = Pending + In Progress + Rejected. Sorted by pickup date (soonest first). Rejected rows are highlighted red and bubble to top. The Status column shows each job's state with color-coded chips. The 4 pills at top tell you where to focus: Active (the work), Completed (needs verification), Verified (done), All (everything).

View 2: Completed (Needs Verification)

Admin action queue
Jobs where proof has been uploaded and an admin needs to review + verify. This is the billing gate — clicking "Verify" on a job marks it as verified and ready for payment. Always a small, actionable list. The "Completed" status chip uses amber to distinguish from both "In Progress" (blue, still working) and "Verified" (green, fully done).

Final Design: Locked In

Status lifecycle: Pending → In Progress → Completed (proof uploaded) → Verified (admin sign-off). Rejected + Cancelled as terminal states. Replaces the old dual status/proofStatus model with one unified field.

4 status colors:
• Gray (Pending) — not started
• Blue (In Progress) — work underway, matches Future Blue accent
• Amber (Completed) — proof uploaded, needs verification. Amber = "action needed" without being alarming
• Green (Verified) — done, ready for billing
• Red (Rejected) — vendor can't fulfill, needs reassignment

Nav pills: Active (Pending + In Progress + Rejected) | Completed | Verified | All. These are the task queues — Active is the work, Completed is the verification queue, Verified is the archive.

Daily workflow:
1. Open Jobs → Active view. Rejected jobs at top (red rows). Handle fires.
2. Monitor In Progress jobs (blue). Check vendors are on track.
3. Click Completed → 9 jobs with proof. Review + Verify each one.
4. Toggle Group by: Vendor → chase vendors about Pending/In Progress jobs.
5. Filter FM + today → morning dispatch view.

Key interactions (unchanged): Click job row → slide-out panel. Click DO link → Delivery Orders view. Group headers collapsible. Vendor column hidden when grouped by vendor. Vendor dropdown with search for 30+ vendors.

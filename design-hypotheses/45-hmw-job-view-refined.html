HMW-45: Job View — Power Table with Group By

"How might we combine the flexibility of a flat power table with the user-friendliness of vendor grouping for proof chasing?"
Refinement of HMW-44. The flat Power Table (Option A) handles all 6 use cases but lacks the visual "progress tracking" of the Vendor Workboard (Option C). The Workboard feels great for proof chasing but breaks for service/time queries. Solution: one table with a "Group by" dropdown — defaults to Vendor grouping (the proof chasing workflow), toggles to flat or service/date grouping when the question changes.

View 1: Group by Vendor (Default)

Default on page load
The proof-chasing view. Jobs grouped under vendor headers with summary badges. Captures Option C's "vendor checklist" feel — scan headers to see who needs attention, expand to see their jobs. No sidebar eating horizontal space. Proof filter is set to show non-validated jobs (the work that needs doing).

Decision: Power Table with Group By toggle (default: Vendor)

One page, one table, one dropdown that changes the lens. Group by: Vendor is the default because proof chasing is the #1 daily task — ops opens the page and immediately sees each vendor's outstanding work with summary badges. When the question changes to "What FM jobs need dispatch?" or "How are all EC clearances?", toggle to Group by: None or Group by: Service. All filters (service pills, proof status, vendor dropdown, date) work identically across all grouping modes.

Key interactions:
• Click any job row → opens the same slide-out panel (proof upload, fee management, activity log)
• Click DO link → navigates to Delivery Orders view with that order expanded
• Group headers are collapsible — collapse vendors you've already handled
• Vendor column hidden when grouped by vendor (redundant), shown when grouped by service/date/none
• Service column hidden when grouped by service, shown otherwise

Default filter: Proof = "Active" (showing awaiting + uploaded + rejected, hiding validated). This is the "work to do" view. Switch to "All" to see validated jobs for billing reconciliation.

H14: Vendor Filter

"How might we let ops find all jobs assigned to a specific vendor -- for coordination, performance review, or capacity planning?"
Problem: Currently the filter bar only supports MAWB search and customer dropdown. Ops often needs to answer questions like "show me all HaleSun jobs today" to coordinate pickups, or "how many jobs does Gonda have this week" for capacity planning. Right now, they scroll through 50+ rows, mentally filtering vendor names. For vendor coordination calls, this wastes 5-10 minutes per session.

Option A: Vendor Dropdown in Filter Bar

Recommended
How it works: Add a "Vendor" dropdown as a third filter, next to the existing MAWB search and Customer dropdown. Selecting a vendor filters the table to show only orders that have at least one job assigned to that vendor. The selected vendor is highlighted in the dropdown with teal styling.
Pros
Fits the existing filter pattern -- consistent, learnable
Simple to implement -- same dropdown pattern as Customer filter
Can combine filters: "HaleSun + TikTok" = HaleSun jobs for TikTok
Clear state: dropdown shows what's filtered
Cons
Filter bar gets wider with a third dropdown
Vendor list could be long (20+ vendors) -- needs search in dropdown
Only filters at the order level, not individual job level

Option B: Clickable Vendor Names in Table

Direct manipulation
How it works: Click any vendor name in the table (in the vendor column or in job chips) to instantly filter the table to that vendor. A "filtered by: HaleSun x" chip appears above the table. Click the "x" to clear the filter. The vendor names act as interactive links (teal color, pointer cursor).
Pros
Fastest path -- one click on any vendor name in the table
Feels intuitive -- "click what you want to see more of"
No filter bar clutter -- the table IS the filter
Works from any row without opening dropdowns
Cons
Non-standard -- clickable cell text is unexpected in a data table
Conflicts with row-click (expand) interaction
No way to discover the feature without trying it
Hard to combine with other filters (customer + vendor)

Option C: Vendor View Tab (Grouped View)

Separate perspective
How it works: A "By Vendor" tab above the table groups all jobs by vendor. Each vendor section shows a header with the vendor name and job count, followed by their assigned jobs across all orders. This gives a vendor-centric view of the workload.
Pros
Complete vendor-centric view -- see all jobs for each vendor
Great for capacity planning ("HaleSun has 5 jobs, Gonda has 2")
Natural for vendor coordination calls
Could show vendor stats (completion rate, total cargo weight)
Cons
A whole new view to build and maintain
Context switch -- user leaves the order-centric table
Doesn't compose with existing MAWB/customer filters
Overkill for Phase 1 -- a filter covers 80% of the use case

Recommendation: Option A -- Vendor Dropdown in Filter Bar

The vendor dropdown is the simplest addition that covers the core need: "show me all jobs for vendor X." It fits the existing filter pattern (MAWB search + Customer dropdown), so ops doesn't need to learn a new interaction. It composes naturally with existing filters -- "HaleSun + TikTok" answers "what HaleSun jobs do we have for TikTok?" Option B (clickable vendor names) is a nice shortcut but breaks the standard table interaction model and is hard to discover. Option C (vendor view tab) is valuable for capacity planning but is a Phase 2 feature -- it requires a new view, new grouping logic, and new summary stats. The dropdown covers 80% of the use case with 20% of the effort.

HMW: Data Density for Logistics Users

"How might we make the UI feel like a smart spreadsheet — dense enough for ops who live in Excel, but with the clarity of intentional design?"

Ops planners manage 50+ delivery orders daily in Excel. They're used to seeing 30-40 rows at a glance with zero wasted space. Our current design shows 5-6 orders per viewport with big cards, generous padding, and 32px dashboard numbers. That's the opposite of their mental model.

Current (SaaS-y)

  • Dashboard: 4 big cards, ~120px height, 32px numbers
  • Table padding: 14px 16px per cell
  • Card radius: 14px (bubbly)
  • Row height: ~70px
  • Visible orders: ~5-6 per viewport
  • Feels like: Modern SaaS template

Proposed (Data-dense)

  • Dashboard: single stats bar, ~32px height
  • Table padding: 8px 12px per cell
  • Card radius: 4-6px (sharp)
  • Row height: ~40px
  • Visible orders: ~12-15 per viewport
  • Feels like: Smart spreadsheet with personality

Key differences from current design

  • Navbar: 40px (was 52px) — 12px saved. Smaller logo, tighter nav links.
  • Dashboard → Stats bar: One line with counts + separators. No cards, no icons, no big numbers. Saves ~120px vertical space.
  • Table padding: 8px 12px (was 14px 16px) — each row saves ~12px height.
  • Border radius: 4-6px (was 14px) — feels sharp and professional, not bubbly.
  • Font sizes: 12-13px for most text. Customer name 13px (was 14px). Chips 10px.
  • Customer + ID on same cell with ID below in smaller text — saves a column vs separate display.
  • Cargo column added (bags + weight) — keeps metadata visible without expanding.
  • ~8 rows visible without scrolling (was ~5). With stats bar and denser rows, ops sees almost 2x more data.
  • Expanded job cards also tighter: 10px padding, 6px radius, smaller text.
  • Filter bar is inline — horizontal with minimal labels, not a full card.

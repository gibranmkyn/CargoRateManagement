H13: Quick Duplicate Orders

"How might we let ops create similar orders in seconds -- since many follow the same vendor/route pattern?"
Problem: Ops creates 10-50 orders/day. Many go to the same warehouses with the same vendors. For example, TikTok ships daily from Dongguan via HaleSun for first-mile trucking. Creating each order from scratch -- selecting customer, adding jobs, picking vendors, entering routes -- takes 2-3 minutes per order. Over 30 orders, that is over an hour of repetitive form-filling.

Option A: "Duplicate" Button on Each Order Row

Recommended
How it works: A small duplicate icon appears on hover at the end of each row. Click it to create a new order pre-filled with the same customer, jobs, routes, and vendors. MAWB and dates are left blank (must be unique per order). The user lands on the create form with all fields pre-populated, ready to adjust and submit.
Pros
One click from the table -- fastest path to duplication
No navigation needed -- hover, click, done
Hidden on non-hover keeps the table clean
Pre-fills everything except what must be unique (MAWB, dates)
Cons
Hover-only means mobile/touch can't access it easily
Accidental clicks could create unwanted draft orders
The button competes with row-click (expand) interaction

Option B: "Create Similar" in Slide-Out Panel

Contextual action
How it works: When viewing a job in the slide-out panel, a "Create similar order" link appears at the bottom. Clicking it opens the create form pre-filled with the parent order's customer, vendor, service, and route details. MAWB and dates are blank.
Pros
Contextual -- user is already looking at the order details
No accidental clicks -- requires deliberate navigation to the panel
Works on touch devices (no hover dependency)
Can show a summary of what will be duplicated before navigating
Cons
Requires 2 clicks: expand row + click link in panel
Slower path than a table-level action
User must open the panel even when they already know they want to duplicate
Link at the bottom of the panel may be below the fold

Option C: Recent Orders Quick-Pick on Create Page

Template-based approach
How it works: On the create order page, a "Recent orders" section at the top shows the last 5 orders. Click any order to pre-fill the form with that order's pattern (customer, vendors, routes, services). The user can then modify any field before submitting.
Pros
Visible on the create page -- natural workflow location
Shows multiple templates at once -- not limited to one order
Could evolve into saved templates / favorites
Works on all devices -- no hover needed
Cons
User must navigate to create page first -- extra step
Recent orders may not be the right templates (what if today's pattern differs)
Takes vertical space on the create page
More complex to build than a simple duplicate button

Recommendation: Option A -- Duplicate Button on Row Hover

The hover duplicate button offers the fastest path: one click from the table, no navigation needed. Ops scanning the table already sees the order they want to copy -- they hover, click "Duplicate," and land on a pre-filled create form. This saves the most time compared to Option B (2 clicks + panel navigation) or Option C (navigate to create page + select template). The hover-only visibility keeps the table clean. For touch/mobile support, we can add the duplicate action to the row's context menu or the slide-out panel as a fallback (borrowing from Option B), but the primary desktop workflow should be the fastest path.

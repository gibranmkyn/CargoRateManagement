HMW-40: FTL Bulk Rate Management via CSV

How might we let rates admins bulk manage FTL trucking rates via CSV upload/download — matching the Excel workflow they already use — while the web UI serves as a read-only dashboard for viewing and verifying rates?
The real workflow today: Vendors send rate cards as Excel spreadsheets. Ops admin receives the Excel, reviews it, and wants to load it into the system. Making them manually type 164 routes × 8 truck types = 1,300+ cells is not viable.

The insight: The web grid should be for viewing and verifying, not for data entry. CSV upload/download is the data entry mechanism. The grid is the dashboard.

The flow:
  1. Download CSV template (or download current rates as CSV)
  2. Edit in Excel (paste vendor's rate card data)
  3. Upload CSV → system validates, shows preview of changes
  4. Confirm → rates are created/updated in bulk
  5. View the rate sheet grid to verify everything looks right

Design Decisions

  • Rate sheet grid = read-only dashboard. View and verify, not data entry. Search + filter within grid rows.
  • CSV is the primary editing mechanism. Download current rates → edit in Excel → re-upload. Matches the vendor rate card workflow.
  • CSV format = vendor, origin_city, origin_district, origin_code (GB/T), destination, dest_code, currency, 1.5T, 3T, 5T, 8T, 10T, 12T, 40HQ, 45HQ. Empty cells = not available.
  • Upload preview shows change summary: new / updated / unchanged / errors. Diff view with old vs new values.
  • Validation checks: origin_code exists in Regions master data, dest_code exists, rate values are numbers, currency is valid, vendor exists.
  • Partial apply: Valid rows can be applied even if some rows have errors.
  • Download options: "Blank template" (column headers only) or "Current rates" (pre-filled with existing data for editing).
  • Destination dropdown above the grid filters by destination district. Supports vendors with multiple hubs.
  • District search above grid rows for finding specific routes in a 92-row table.

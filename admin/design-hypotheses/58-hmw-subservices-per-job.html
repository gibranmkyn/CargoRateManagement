HMW-58: L2 Subservices per Job

How might we let ops planners capture which L2 subservices are relevant to each job, so trip data can be exported at the subservice level?
Context: Jobs have 1 L1 service but need multiple L2 subservices. Hierarchy: CR(2), CS(6), EC(3), FM(3), OH(5). Export = 1 row per job × subservice. All unchecked by default — planner explicitly checks what applies.

Decision

Scrollable Checklist — all unchecked by default

Pattern: Fixed-height (~80px) scrollable box with checkboxes below each job's vendor/location fields. Scrolls when L2 count exceeds visible space. All unchecked on job creation — planner checks what applies. Each item: checkbox + L2 name + cost ID (mono, right-aligned).

Data model: Add l2CostIds: string[] to Job. Empty by default. Populated from checked items.

Export: 1 CSV row per job × checked L2. Jobs with 0 checked → 0 rows. Added columns: Cost ID, Subservice (L2).

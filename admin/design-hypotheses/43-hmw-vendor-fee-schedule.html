HMW-43: Vendor Fee Schedules

How might we let each vendor have their own fee schedule (not forced into fixed L2 codes), auto-populate all applicable fees on job creation, and let ops remove the exceptions — so managing fees is subtractive, not additive?
What the real vendor rate cards look like:

HaleSun (Origin Handling at Shenzhen Hub): 15+ fees — 前置仓操作费 0.30/kg, 转仓费 100-300/truck, 提单费 50/awb, 仓储费 0.10/kg/day, 装卸费 0.20/kg, 安检费 0.30/kg, 托盘费 120-150/plt, 包装膜 6-8/ctn, 贴标费 1.50/ctn...

Gonda (HKG Warehouse): 10+ fees — 装板服务 1.95/kg (all-in ULD build-up), 提单打单费 150/份, 登记费 350/票, X-Ray 0.80/kg, 仓→货运站运输 0.60/kg...

Key insight: Every vendor has their own fee names, their own units, their own granularity. Forcing them into OH001-OH005 loses information. The system should accept vendor fee schedules as-is.

Second insight: 90% of jobs for the same vendor+location need the same fees. Don't make ops select 5 fees 50 times/day. Auto-populate all, let ops remove exceptions.

Part 3: The Full Picture

Rate card (admin uploads via CSV):
Vendor + Service + Location → list of fees (name, unit, rate, min charge)
Each fee is the vendor's own term. No forced L2 mapping.
CSV format: fee_name, unit, rate, min_charge, currency
Job creation (auto-populate):
Ops picks vendor + service + location → all fees from rate card auto-populate
Each fee starts as ✓ included with qty auto-calculated from order data
Ops removes exceptions (click ✓ to toggle off)
Ops edits quantities where needed
Slide-out panel (during job lifecycle):
Shows active fees with locked rates + editable quantities
Removed fees shown as muted line-through (can re-add with + click)
Rate is locked from card, only qty changes
When job is validated → all fees lock (read-only)
What this replaces:
~~Fixed L2 Cost IDs (OH001-OH005)~~ → vendor-specific fee names
~~Rigid 20 sub-services~~ → flexible per-vendor fee schedules
~~"All fees auto-populate, can't remove"~~ → "All fees auto-populate, remove exceptions"

Design Decisions

  • Rate card = vendor's own fee schedule. No forced mapping to standard L2 codes. Each vendor can have their own fee names, units, and granularity.
  • Fee schedule is per vendor + service + location. HaleSun's OH fees at Shenzhen WH are different from Gonda's OH fees at HKG Warehouse.
  • CSV upload for fee schedules. Format: fee_name, unit, rate, min_charge, currency. Same simple pattern as FTL trucking CSV.
  • Auto-populate all on job creation. When a job is created for vendor+service+location, all fees from the schedule appear as active fee line items.
  • Subtractive: remove exceptions. Ops clicks ✓ to toggle a fee off. Removed fees show as muted line-through (can re-add). This is faster than selecting from a menu.
  • Quantity editing. Auto-calculated from order data (/kg → weight, /ctn → carton count, /plt → pallet count, /awb → 1). Ops can override.
  • Minimum charges. The rate card supports min_charge per fee. If qty × rate < min, the min applies.
  • L2 Cost IDs become optional tags. Admin CAN tag a fee with a Cost ID for reporting/billing. But it's not required and doesn't constrain the fee name or unit.

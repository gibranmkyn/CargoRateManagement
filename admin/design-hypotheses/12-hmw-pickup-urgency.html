H12: Pickup Urgency

"How might we help ops see at a glance which orders are picking up soon vs. days away?"
Problem: The pickup date column shows dates but doesn't convey urgency. An order picking up in 45 minutes should feel different from one 3 days out. Right now, ops has to mentally parse "25 Mar, 14:30" and compare it against the current time. When scanning 50 rows, this mental math adds up. Critical pickups get missed because a "26 Mar" date doesn't scream "that's in 90 minutes."

Option A: Color-Coded Pickup Time

Recommended (combined with C)
How it works: The pickup date text uses color to signal urgency. Less than 2 hours: red text (#dc2626) -- urgent. 2-24 hours: default text (#111827) -- today. More than 24 hours: muted text (#9ca3af) -- not urgent. The date format stays the same; only the color changes.
Pros
Zero layout change -- just text color
Instant pattern recognition: red = act now
Consistent with 3-color system (red = urgent)
Works with any date format
Cons
Ops still needs to read the date to know exactly when
Red text might be confused with "rejected" status
Colorblind users may miss the signal

Option B: Countdown Badge

Relative time display
How it works: Replace the absolute date with a relative countdown: "45m", "3h", "2d". Color-coded: red for < 2h, default for same day, muted for future days. The absolute date appears on hover as a tooltip.
Pros
Immediate comprehension -- "45m" is faster than parsing a date
Compact -- saves horizontal space
Color + text reinforces urgency double
Very common pattern in logistics dashboards
Cons
Loses the absolute date at a glance (need hover)
Countdown changes constantly -- "45m" becomes "44m" a minute later
Requires real-time clock updates (timer logic)
Ops may want to see the actual date for coordination calls

Option C: Sort by Urgency by Default

Structural change
How it works: The table is auto-sorted by pickup date (soonest first) as the default sort order. No visual change to the cells themselves -- the soonest pickup is always at the top. Combined with Option A, the first few rows are red-text urgent pickups, then default-text today pickups, then muted future ones.
Pros
Zero visual noise -- just reorders what's already there
The most urgent order is always at the top of the list
Combined with color-coding, creates a natural urgency gradient
Ops can override by clicking another column header
Cons
By itself, not enough -- "28 Mar" at row 3 doesn't say "relax"
Ops may prefer to sort by customer or MAWB for other tasks
Row positions shift as time passes -- can be disorienting

Recommendation: Option A + C Combined -- Color-Coded Dates + Default Urgency Sort

The combination of color-coded pickup text (Option A) and default sort by soonest pickup (Option C) delivers the biggest impact with the smallest change. The table opens with the most urgent orders on top, and the red/default/muted text coloring creates a natural visual gradient -- red rows at the top demand action, muted rows at the bottom can wait. Option B's countdown badges are tempting but lose the absolute date that ops needs when coordinating with vendors on the phone ("pickup is at 14:30" not "pickup is in 45 minutes"). The countdown can be explored as a Phase 2 enhancement.

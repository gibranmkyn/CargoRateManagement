HMW-64: JobSlideOut verification interaction — simplification

How might we let ops verify or reject a job without stacking three colored containers and multiple filled buttons in the slide-out?
Problem: The current Verification Action Bar (built in Phase 2 chunk 2c) stacks a second filled colored container below the Status Action Bar. Each variant (Pending / Verified / Rejected) has 2–3 filled buttons. The reject form adds a third red container. That's too many colors competing for attention when the user is making ONE decision.

Design memory: "Simple design users understand beats complex design that preserves more signals" (feedback 2026-04-20). Also: "one dominant pill per row" — the same principle should extend to stacked filled containers within a panel.

Proposed resolution

Adopt the "After" design. Replace the filled Verification Action Bar with a text-only inline row separated by a 1px top divider. The primary action (Verify) keeps a filled green pill so the user's main choice remains obvious. Reject is downgraded to a ghost-with-red-text affordance. The Reject form drops its red container and becomes a plain textarea; the red only returns on the Confirm button at the moment the user is about to commit.

Net effect: Panel top section has exactly ONE filled colored container (the existing Status Action Bar) across every state. Verification state is communicated with a single colored dot + label — same rule we already shipped in the Jobs table (HMW-60).

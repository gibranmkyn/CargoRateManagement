HMW-27: Simplified Sub-Table

How might we simplify the delivery orders sub-table so it's fast to scan, with fee details available only in the slide-out — and remove the priority tag feature?
What changed: The current sub-table has expandable job rows that reveal an L2 fee breakdown table — 3 levels of nesting (order → jobs → fees). This is too much depth for daily scanning. Fee details are already in the slide-out panel. Priority tags add visual noise for an edge case. Both should be removed to keep the table focused on the most common ops workflow: scan jobs, check status, click to act.

Option A: Flat Sub-Table — Click Row Opens Slide-Out

Recommended
Thesis: Sub-table rows are flat (no expand). Total Cost column shows the fee sum. Clicking any row opens the slide-out which has the full fee breakdown. This is how the app worked before the fee expansion was added — but now with the Total Cost column showing the aggregate. Simplest, fastest to scan.
✓ Pros
• 2 levels only (order → jobs) — fast to scan
• One click action: click row = open slide-out with all details
• Total Cost gives the number ops needs at a glance
• Fee breakdown is already excellent in the slide-out
✗ Cons
• Can't see fee details without opening slide-out
• Comparing fees across jobs requires opening each one

Option B: Flat Sub-Table + Cost Tooltip on Hover

Middle Ground
Thesis: Same flat sub-table as Option A. But hovering over the Total Cost cell shows a small tooltip with the fee breakdown — no click required. The slide-out still has the full editable breakdown.
✓ Pros
• Quick peek at fee breakdown without opening slide-out
• Still flat and scannable
• Doesn't clutter the table
✗ Cons
• Hover tooltips aren't discoverable — ops may never find this
• Tooltips are read-only — can't edit fees there
• More implementation complexity for marginal value

Verdict: Option A — Flat Sub-Table, No Inline Fees

Recommended because: The sub-table's job is scanning — "which jobs, what status, how much." The slide-out's job is acting — "change status, view fees, edit quantities, upload proofs." Mixing these (inline fee expansion) blurs the boundary. Option A keeps each surface doing one thing well.

Changes from current design:

  • Remove: Expandable fee rows (L2 breakdown) from sub-table
  • Remove: Expand chevrons (▸/▾) on job rows
  • Remove: expandedJobId state
  • Remove: Priority tag on orders (overdesign for current scope)
  • Keep: Total Cost column (sum of fees)
  • Keep: Click row → opens slide-out with full fee breakdown
  • Keep: Order Total at bottom of expanded sub-table

H15: Proof Document Preview

"How might we let ops verify proof documents at a glance without opening the slide-out panel?"
Problem: The sub-table shows "1 doc" or "0 docs" in the Proofs column. Ops needs to verify proofs quickly -- especially for billing readiness checks. Currently, ops has to click each job row to open the slide-out panel just to see if a proof was uploaded or what the file name is. When checking 15 completed jobs for billing, that is 15 panel opens and closes. The question is usually simple: "does this job have a proof or not?"

Option A: Tooltip Preview on Hover

Rich preview
How it works: Hover over "1 doc" in the proofs column to see a tooltip showing the file name and upload date. For image proofs, the tooltip could show a small thumbnail. The tooltip appears instantly on hover and disappears on mouse-out.
Pros
Rich detail without opening the panel
File name visible for verification
Thumbnail preview for image proofs is very useful
No layout changes to the table
Cons
Hover-only -- doesn't work on touch devices
Tooltip can obstruct nearby rows
Requires loading file metadata on hover (latency)
"1 doc" text alone doesn't signal presence/absence clearly

Option B: Inline File Names in Proofs Column

Detailed inline
How it works: Instead of "1 doc", show the actual filename truncated to fit the column: "POD-tiktok..." Multiple files show the first file plus a count: "POD-tik... +1". No hover needed -- the information is always visible.
Pros
Always visible -- no hover or click needed
File name helps ops verify the right document was uploaded
"+1" count shows if multiple proofs exist
Works on touch devices
Cons
Truncated file names may not be useful ("POD-tik..." doesn't help much)
Column needs to be wider to show file names
Visual noise -- every proof cell has different-length text
For billing checks, file name doesn't matter -- just "is there a proof?"

Option C: Proof Status Icon

Recommended
How it works: Replace "N docs" with a simple icon. Green check mark = proof uploaded. Gray circle = no proof. This aligns with the 3-color system: green for completed/present, gray for missing/pending. The icon is compact and scannable. Ops just needs to answer one question: "is there a proof or not?" The file details remain available in the slide-out panel.
Pros
Most compact -- takes minimal column width
Aligns perfectly with 3-color status system
Instantly scannable -- green checks stand out, gray circles blend
Answers the real question: "does this job have proof?"
File details stay in the slide-out (where they belong for deep inspection)
Cons
Loses "how many" docs info (1 doc vs 3 docs)
No file name preview without opening panel
Might feel too minimal for power users who want more detail

Recommendation: Option C -- Proof Status Icon

The icon approach is the most compact and aligns naturally with the 3-color system already used throughout the UI. Green check = proof present (done). Gray circle = proof missing (needs attention). When ops scans the sub-table during a billing readiness check, they are asking a binary question: "does this job have proof?" -- not "what's the filename?" The icon answers that instantly. A column of green checks and gray circles is scannable in under a second, whereas reading "1 doc" / "0 docs" or parsing truncated filenames takes measurably longer. The file details (name, upload date, thumbnail) belong in the slide-out panel for the cases when ops actually needs to inspect the document, which is a separate, less frequent task.

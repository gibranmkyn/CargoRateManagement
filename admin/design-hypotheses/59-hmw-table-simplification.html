HMW-59 · Table Simplification

How might we simplify the Trips and Jobs tables so ops planners can scan at a glance without feeling overwhelmed?

Both tables have accumulated columns beyond their designed intent. Trips grew to 9 data columns (designed: 7). Jobs grew to 9 (designed: 6). The question: how far do we consolidate?

HMW: Simplify Trips + Jobs tables to their essential columns — reduce visual noise, restore scanning speed, eliminate inconsistency between pages.
Option A — Column Consolidation (implement design intent)
Consolidate to designed column set
Merge Origin + Destination → Route. In Jobs: combine Trip ID + Job ID + Customer into one cell. Secondary info (MAWB) becomes a sub-line. Both tables drop from 9 → 7 columns. Aligns with HMW-52 and DESIGN.md spec already decided.

Recommendation
Option A — Column Consolidation
The tables feel overwhelming not because they're "fancy" — the actual styling is already clean — but because they have too many columns (9 each) with redundant or splittable information. Option A reduces both to 7 by implementing what was already decided: HMW-52 called for Route as one column in Trips, and DESIGN.md specifies 6 columns for Jobs. The code diverged from those decisions. Option A is the fix.

Option B is too aggressive — removing Pickup date from Jobs and Customer from Jobs loses meaningful scanning context ops planners need. Option C doesn't actually address the root cause.
Specific changes to implement
TRIPS Merge Origin + Destination → single Route column ("Guangzhou → KLIA2"). Remove separate Origin and Destination columns.
TRIPS Move MAWB as secondary line under Trip ID cell. Remove standalone MAWB column. Trip cell becomes: Trip ID on line 1, MAWB in mono muted on line 2.
TRIPS KEEP Keep Pickup, Delivery, Load, Verification columns as-is. These are all useful at-a-glance.
JOBS Remove separate Trip ID column and Job ID column. Combine into one Trip · Job cell: "T00001 · J03" in mono, muted. Keep as navigation anchor (clicking Trip goes to Trips view).
JOBS Merge Origin + Destination → Route cell ("Shenzhen → KLIA"). Remove separate origin and destination columns.
JOBS KEEP Keep Customer, Vendor, Service, Pickup, Status columns. Customer belongs in Jobs — ops needs to know which client's job this is without expanding.
REMOVE Remove the Trip ID badge/chip styling in Jobs page. Plain mono text ("T00001") is enough — the chip adds visual weight without benefit.
REMOVE Remove Stats bar from both pages. The filter pills already show the same counts (active/completed/verified/all). The stats bar is pure duplication — 32px of redundant data that adds a layer before the page header.

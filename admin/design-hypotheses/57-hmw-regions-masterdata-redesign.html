HMW-57: Regions Master Data Redesign

How might we make the China GB/T 2260 hierarchy browser readable and useful for ops admins who look up district codes during facility setup — without the clutter, dead columns, and ugly indentation of the current table?
Context: The Regions tab shows 33 provinces → 344 cities → 3,077 districts from the GB/T 2260 standard. It's a reference table — admins look things up, they don't create or edit. The current design has 4 pain points: (1) "English" column is empty for nearly all city/district rows; (2) "Level" column badges are redundant with indentation; (3) "Count" column has different meanings at each level; (4) chevron symbols ▾/▸ look dated and the indent is inconsistent across columns. Result: a table full of hollow cells that looks broken.

Option A — Slim 3-Column Table

Recommended
Approach: Drop the English and Level columns entirely. Merge everything into 3 columns: Name (with unified indent line), Code, and a smart Count that adapts per level. Province rows get a subtle left accent. Chevrons become a proper SVG icon inside the Name cell itself — no separate column. The hierarchy is expressed purely through indentation, weight, and color — not separate classification badges.
Pros
· Zero empty cells — every column has data at every level
· Hierarchy instantly readable via weight + indent
· Consistent with Facilities tab table structure
· Clean monospace codes stand out clearly
Cons
· No explicit "Level" label — must infer from visual hierarchy
· Province code and district code look the same without context

Option B — Province-as-Section-Header

Strong Candidate
Approach: Province rows become section dividers (not table rows) — full-width dark band with province name, code, and counts on one line. The table below each province only contains cities + their collapsible districts. This matches the Trips/Jobs page pattern where a parent row acts as a collapsible header for child rows. Level is now structurally obvious, not a badge.
Pros
· Province visually dominates — hierarchy unmistakable
· City rows are the "real" table — clean columns, no indentation needed
· Districts collapse cleanly as sub-rows, just like job sub-table
· Consistent with patterns already in the app
Cons
· Province header is outside the table — can't sort or filter by province column
· Scrolling through 33 provinces with sections is verbose
· Doesn't reduce the vertical height for expanded trees

Option C — Column Explorer (Finder-style)

Radical
Approach: Eliminate the tree table entirely. Replace with a 3-panel column browser (like macOS Finder): Province list on the left, Cities in the middle, Districts on the right. Clicking a province shows its cities. Clicking a city shows its districts. No expand/collapse — the hierarchy is spatially separated. Search filters all panels simultaneously.
Pros
· No nesting, no tree depth to manage
· Province → City → District visible at once without scrolling
· Immediate feedback: click province, instantly see its 20+ cities
· District codes read clearly with full context visible
Cons
· Three fixed panels require wide viewport (≥1200px)
· Unfamiliar pattern — ops admins expect tables, not Finder
· Hard to print/export visually as a table
· Diverges furthest from the app's existing design language

Recommendation: Option A — Slim 3-Column Table

Option A fixes every visual problem with the minimum code change. Drop the English column (mostly empty), drop the Level column (redundant with hierarchy), merge the chevron into the Name cell (consistent indent across all levels), and rename "Count" to "Cities / Districts" with right-aligned text. The result is zero hollow cells, clean monospace codes, and a left-border accent on expanded provinces.

Option B (Section Headers) looks great and is consistent with the trips sub-table pattern, but breaks the table contract — provinces become div elements outside the table, making scroll and search behavior more complex to implement.

Option C (Column Explorer) is elegant for browsing but departs from the app's table-first language, and 1200px minimum viewport makes it risky. Reserve for a future iteration if ops request it.

Changes required: Remove English column + Level column. Move chevron SVG inside Name cell. Rename column header. Align Count right. Add border-left: 2px solid #152CFF to expanded province rows. Zero behavior change.
